# 糖豆人记账游戏 (SugarRunner Expense) - 产品需求文档（PRD）

---

## 一、项目背景与目标

### 1.1 项目背景

当前市面上的记账软件多为传统表单式交互，用户容易因操作繁琐而放弃记账。本项目将游戏化互动与传统记账功能结合，以糖豆人角色为核心视觉符号，用户通过点击彩色小球配置消费信息、糖豆人吃掉小球完成记账，让记账像玩游戏一样自然轻松。

### 1.2 产品定位

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 糖豆人记账游戏 (SugarRunner Expense) |
| 产品形态 | 桌面端 Web 应用（移动端后续适配） |
| 适配平台 | 第一阶段：桌面浏览器（Chrome / Edge / Firefox）；第二阶段：移动端 |
| 目标用户 | 22-35 岁年轻人群（职场新人、学生、自由职业者） |
| 核心价值 | 游戏化记账 → 自动归类 → 可视化洞察 → 消费可视化 |
| 差异化特点 | 以糖豆人游戏互动替代传统表单记账，降低记账心理门槛 |
| 支持币种 | 人民币(CNY)、美元(USD)、欧元(EUR)、日元(JPY)、英镑(GBP)、港币(HKD) |

### 1.3 项目目标

1. 通过游戏化交互降低用户记账心理门槛
2. 通过可视化报表帮助用户了解消费结构
3. 通过预算设置帮助用户控制开支
4. 通过趣味反馈提升用户记账频率和留存

| 角色 | 描述 |

## 五、业务流程图

### 5.1 用户整体流程

```
[打开页面] → [登录界面（糖豆人吉祥物+表单）]
                ├── 有账号 → [邮箱+密码登录] → [游戏主界面]
                ├── 有账号 → [邮箱+验证码登录] → [游戏主界面]
                ├── 无账号 → [密码注册] → [输入邮箱+密码+确认] → [游戏主界面]
                ├── 无账号 → [验证码注册] → [输入邮箱+验证码] → [游戏主界面]
                └── 忘记密码 → [邮箱验证码重置] → [登录界面]

[游戏主界面: 糖豆人走动 + 彩色小球]
    ├── 点击小球 → [配置面板: 选分类+金额] → [确认] → [糖豆人吃掉小球(动画)] → [余额更新]
    ├── 点击糖豆人 → [右侧面板(约35%宽)]
    │       ├── [今日账单: 按时间倒序列表]
    │       ├── [统计: 饼图 + 消费时间轴]
    │       └── [设置]
    │           ├── [导出数据]
    │           ├── [管理分类]
    │           ├── [币种切换]
    │           ├── [清空数据(二次确认)]
    │           └── [账号注销(可选)]
    └── 糖豆人空闲 → [随机走动/偶尔跳跃]

```

### 5.2 记账核心流程（游戏化）

```
[点击小球] → [浮出毛玻璃配置面板(小球旁边)]
                ├── 点击分类图标 → 高亮选中
                ├── 点击快捷金额 → 高亮选中
                └── 点击确认按钮
                        ↓
              [面板关闭]
                        ↓
              [小球更新: 显示金额数字 + 变大 + 颜色加深]
                        ↓
              [糖豆人检测到已配置小球]
                        ↓
              [糖豆人平滑走向最近已配置小球 (2秒内出发)]
                        ↓
              [小球飞向糖豆人 → 嘴巴张开 → 爆炸星星特效 → 打嗝上跳]
                        ↓
              [账单自动保存 + 余额滚动更新]
                        ↓
              [小球消失]
                        ↓
              [检查剩余小球数: <3 → 自动生成新小球补充]
```

| --- | --- |
| AC-01 | 新用户输入邮箱（格式校验）、密码（≥6 位）、确认密码一致 → 注册成功并自动登录进入游戏主界面 |

## 八、边界条件

| 编号 | 边界条件 | 处理方式 |
| --- | --- | --- |
| BC-01 | 用户未登录直接访问主界面 | 重定向到登录页 |
| BC-02 | 登录态过期 | 下次请求时提示重新登录，跳转登录页 |
| BC-03 | 小球配置面板打开时点击另一个小球 | 关闭当前面板，在新小球旁边打开新面板 |
| BC-04 | 糖豆人正在进食时用户点击新小球配置 | 完成当前进食动画后再走向下一个 |
| BC-05 | 快捷金额均为正数，无需校验负数 | 快捷按钮数值固定为正整数（10/20/50/100/200） |
| BC-06 | 未选择分类或金额就点确认 | 确认按钮置灰不可点击 |
| BC-07 | 网络断开时进行记账 | 提示网络异常，记账失败，小球恢复未配置状态 |
| BC-08 | 浏览器窗口过小（宽度 < 1024px） | 弹出提示"建议使用更大屏幕获得最佳体验"，允许继续使用但布局可能压缩 |
| BC-09 | 分类被删除但有历史账单使用该分类 | 历史账单保留原分类名称，不受影响 |
| BC-10 | 清空数据操作 | 二次确认弹窗，确认后删除当前用户所有数据，不可恢复 |
| BC-11 | 账号注销操作 | 二次确认弹窗，确认后删除账号及所有数据，跳转登录页 |
| BC-12 | 浏览器不支持 localStorage/IndexedDB | 页面加载时检测，不支持则提示"浏览器版本过低，请升级" |
| BC-13 | 验证码发送频率限制 | 60 秒内不可重复发送，前端倒计时展示 |
| BC-14 | 首次登录无预设分类 | 系统自动创建预设分类，确保开箱即用 |
| BC-15 | 小球全部被清空 | 自动检测并补充新小球（剩余<3触发） |
| BC-16 | 游戏主界面打开时切换到后台 | 糖豆人暂停移动，回到前台后恢复 |

## 九、非功能性需求

| 编号 | 类别 | 需求描述 |
| --- | --- | --- |
| NFR-01 | 性能 | 页面首次加载时间 ≤ 3 秒（4G 网络环境） |
| NFR-02 | 性能 | 糖豆人移动动画帧率 ≥ 30fps，无明显卡顿 |
| NFR-03 | 性能 | 小球配置面板弹出动画流畅，关闭响应即时 |
| NFR-04 | 性能 | 进食动画（小球飞行→爆炸星星→打嗝）整体时长 ≤ 1.5 秒 |
| NFR-05 | 兼容性 | 支持 Chrome 最新两个版本、Edge 最新两个版本、Firefox 最新版本（桌面端） |
| NFR-06 | 兼容性 | 推荐分辨率 1366x768 及以上，最低支持 1024x768 |
| NFR-07 | 安全性 | 用户密码加密存储，不可明文保存 |
| NFR-08 | 安全性 | 用户数据按账号隔离，接口需鉴权，不可越权访问 |
| NFR-09 | 安全性 | 验证码有效期 5 分钟，单日单邮箱发送上限 10 次 |
| NFR-10 | 可用性 | 所有可点击交互元素最小点击区域 ≥ 32x32px（桌面端鼠标交互） |
| NFR-11 | 可用性 | 主界面零文字引导，所有操作反馈通过动画和视觉变化传递 |
| NFR-12 | 可用性 | 登录/注册表单错误提示友好清晰，位置贴近对应输入框 |
| NFR-13 | 视觉风格 | 整体可爱治愈温暖风格，暖色渐变背景，圆润无衬线字体 |
| NFR-14 | 数据存储 | 账单记录、余额、分类列表必须持久化存储（浏览器 IndexedDB），刷新不丢失 |
| NFR-15 | 数据存储 | 小球状态不持久化，每次刷新页面重新随机生成 |
| NFR-16 | 国际化 | 支持中文界面，币种以用户选择的货币符号展示 |
| NFR-17 | 后续规划 | 第二阶段适配移动端：响应式布局、触摸交互、底部抽屉面板还原 |
